# Amowebhooks

Amowebhooks — внутренний FastAPI-сервис для обмена данными между amoCRM и внешними системами. Сервис принимает вебхуки, создаёт задачи, контакты и сделки в amoCRM, передаёт аналитические данные в Google Sheets, сохраняет рекламные метки переходов и формирует коммерческие предложения в HTML и PDF.

## Возможности

- обработка заданий и товаров, поступающих из Google Sheets и маркетплейса;
- обработка уведомлений о новых заказах Яндекс Маркета;
- создание контактов, сделок, примечаний и задач в amoCRM;
- выгрузка аналитики по сделкам и покупателям в Google Sheets;
- сохранение UTM-меток, `yclid`, `cm_id` и `block` перед переходом в Telegram- или MAX-бота;
- получение ранее сохранённых UTM-меток по идентификатору перехода;
- создание коммерческих предложений на основе данных сделки amoCRM;
- формирование PDF через установленный Google Chrome или Chromium;
- отправка служебных уведомлений администратору через Telegram-бота.

## Стек и внешние системы

- Python 3.11;
- FastAPI и Uvicorn;
- aiohttp и requests;
- aiogram;
- SQLAlchemy;
- Jinja2;
- Chrome/Chromium в headless-режиме;
- amoCRM API;
- Яндекс Маркет Partner API;
- Google Apps Script или другой HTTP-вебхук Google Sheets.

## Требования

Для локального запуска нужны:

- Python 3.11;
- доступ к учётным данным подключаемых систем;
- Google Chrome или Chromium, если требуется формирование PDF;
- доступная СУБД, указанная в `DATABASE_URL`. По умолчанию используется локальный SQLite-файл `amowebhook.db`.

## Настройка окружения

Настройки загружаются из файла `.env` в корне проекта. Файл содержит секреты и не должен попадать в систему контроля версий, логи или сообщения об ошибках.

| Переменная | Обязательна | Назначение | Значение по умолчанию / безопасный пример |
|---|---:|---|---|
| `BOT_TOKEN` | Да | Токен Telegram-бота для служебных уведомлений | `123456:replace_me` |
| `AMOCRM_SUBDOMAIN` | Да | Поддомен аккаунта amoCRM | `example` |
| `AMOCRM_CLIENT_ID` | Да | ID OAuth-интеграции amoCRM | `replace_me` |
| `AMOCRM_CLIENT_SECRET` | Да | Секрет OAuth-интеграции amoCRM | `replace_me` |
| `AMOCRM_REDIRECT_URL` | Да | Redirect URI интеграции amoCRM | `https://example.test/oauth/callback` |
| `AMOCRM_ACCESS_TOKEN` | Да | Текущий access token amoCRM | `replace_me` |
| `AMOCRM_REFRESH_TOKEN` | Да | Текущий refresh token amoCRM | `replace_me` |
| `AMOCRM_SECRET` | Да | Код авторизации для первичного OAuth-обмена | `replace_me` |
| `ADMIN_ID` | Да | ID Telegram-чата администратора | `123456789` |
| `YANDEX_API` | Да | API-ключ Яндекс Маркета | `replace_me` |
| `MAGAZINE_ID` | Да | ID кампании магазина в Яндекс Маркете | `123456` |
| `GOOGLE_SHEETS_WEBHOOK_URL` | Нет | URL обработчика выгрузки аналитики по сделкам | `None` |
| `GOOGLE_SHEETS_CUSTOMERS_WEBHOOK_URL` | Нет | URL обработчика выгрузки аналитики по покупателям | `None` |
| `GOOGLE_SHEETS_TOKEN` | Нет | Общий токен защиты аналитических маршрутов | `None` |
| `DATABASE_URL` | Нет | SQLAlchemy URL базы данных | `sqlite:///./amowebhook.db` |
| `TELEGRAM_BOT_URL` | Нет | Ссылка для перенаправления в Telegram-бота | `https://t.me/your_bot` |
| `MAX_BOT_URL` | Да | Ссылка для перенаправления в MAX-бота | `https://example.test/max-bot` |
| `GET_UTM_TOKEN` | Нет | Токен защиты маршрута получения UTM-меток | `None` |

Несмотря на аннотации `str | None`, переменные `AMOCRM_ACCESS_TOKEN` и `AMOCRM_REFRESH_TOKEN` читаются без значения по умолчанию и должны присутствовать в `.env`. При обновлении OAuth-токенов amoCRM сервис записывает новые значения этих переменных обратно в `.env`, поэтому файл должен быть доступен процессу на запись.

Минимальная структура файла:

```dotenv
BOT_TOKEN=123456:replace_me
AMOCRM_SUBDOMAIN=example
AMOCRM_CLIENT_ID=replace_me
AMOCRM_CLIENT_SECRET=replace_me
AMOCRM_REDIRECT_URL=https://example.test/oauth/callback
AMOCRM_ACCESS_TOKEN=replace_me
AMOCRM_REFRESH_TOKEN=replace_me
AMOCRM_SECRET=replace_me
ADMIN_ID=123456789
YANDEX_API=replace_me
MAGAZINE_ID=123456
MAX_BOT_URL=https://example.test/max-bot
```

Переменные Google Sheets и `GET_UTM_TOKEN` формально необязательны для загрузки конфигурации, но соответствующие маршруты без них возвращают ошибку конфигурации. Для SQLite переменную `DATABASE_URL` можно не задавать.

## Запуск

Запускайте приложение из корня проекта, поскольку пути к шаблонам и `.env` задаются относительно текущего каталога:

```bash
uvicorn main:app --reload
```

По умолчанию сервис доступен по адресу `http://127.0.0.1:8000`:

- Swagger UI: `http://127.0.0.1:8000/docs`;
- ReDoc: `http://127.0.0.1:8000/redoc`.

При старте SQLAlchemy создаёт отсутствующие таблицы через `Base.metadata.create_all()`, после чего открывается HTTP-сессия клиента amoCRM. При остановке приложения эта сессия закрывается.

Для production-запуска отключите `--reload`, ограничьте доступ к служебным маршрутам на уровне reverse proxy и передавайте секреты через защищённое окружение. Проект не содержит готовой конфигурации Docker, systemd или конкретной облачной платформы.

### Хранение рекламных меток

Маршруты `/telegram` и `/max` собирают метки из query-параметров, cookies и cookies Sourcebuster. Значения выбираются в следующем порядке: query-параметр, обычная cookie, `sbjs_current`, затем `sbjs_first`.

Каждый переход создаёт строку в таблице `education_visits` со следующими полями:

- `utm_source`, `utm_medium`, `utm_campaign`, `utm_content`, `utm_term`;
- `yclid`, `cm_id`, `block`;
- время создания записи.

ID записи передаётся целевому боту как параметр `start`. Дедупликация переходов по `yclid` в текущей реализации отключена.

## HTTP API

Маршруты ниже описывают фактическое поведение текущей реализации. У вебхуков нет отдельной проверки подписи или токена, если это явно не указано в описании маршрута.

### Аналитика и Google Sheets

| Метод и маршрут | Входные данные | Результат и побочные эффекты | Основные ошибки |
|---|---|---|---|
| `GET /analyze` | Query: `token`, `request_id` | Возвращает `202`-подобный по смыслу ответ `{"status":"accepted"...}` с HTTP 200 и запускает фоновую выгрузку аналитики по сделкам | `401` при неверном токене; `500`, если не заданы URL или токен Google Sheets |
| `GET /analyze_customers` | Query: `token`, `request_id` | Ставит в фон выгрузку сводной аналитики по покупателям и их сделкам, возвращает HTTP 200 | `401` при неверном токене; `500`, если отсутствуют основной URL Google Sheets или токен; ошибка фоновой задачи при недоступном URL аналитики покупателей |

`request_id` передаётся внешнему обработчику и используется для сопоставления запроса с результатом. Фактическая отправка выполняется после возврата HTTP-ответа, поэтому успешный ответ означает принятие задачи, а не успешное завершение выгрузки.

Пример:

```bash
curl "http://127.0.0.1:8000/analyze?token=replace_me&request_id=report-2026-07-14"
```

### Вебхуки

| Метод и маршрут | Входные данные | Результат и побочные эффекты | Основные ошибки |
|---|---|---|---|
| `POST /sheets` | JSON: `timestamp`, `phone`, `fullName`, `description`, необязательный `materialsLink` | Ищет контакт по телефону и создаёт задачу в amoCRM; возвращает `{"status":"ok"}` | Ошибка формата даты; ошибка amoCRM; исключение и Telegram-уведомление, если контакт не найден |
| `POST /sheets/marketplace` | JSON: `data.lead_id`, `data.items[]`; у товара используется `quantity` | Отбрасывает позиции с количеством меньше 1 и добавляет остальные элементы каталога к сделке amoCRM | Ошибка преобразования ID/количества или ошибка amoCRM |
| `POST /market/new_order/notification` | JSON: `orderId` | Уведомляет администратора, получает заказ и покупателя из Яндекс Маркета, создаёт контакт, сделку и примечание в amoCRM | Исключения журналируются, но маршрут всё равно возвращает служебный JSON из блока `finally` |
| `POST /new_message_tp` | JSON, form-urlencoded, текст или пустое тело | Разбирает тело запроса и возвращает `{"status":"ok"}`; дальнейшая обработка сейчас отсутствует | Стандартные ошибки чтения запроса |

Дата для `/sheets` должна иметь формат `ДД.ММ.ГГГГ ЧЧ:ММ:СС`. Перед созданием задачи сервис прибавляет к ней два часа.

Пример создания задачи:

```bash
curl -X POST "http://127.0.0.1:8000/sheets" \
  -H "Content-Type: application/json" \
  -d '{
    "timestamp": "14.07.2026 12:30:00",
    "phone": "+7 900 000-00-00",
    "fullName": "Тестовый пользователь",
    "description": "Связаться с клиентом",
    "materialsLink": "https://example.test/materials"
  }'
```

Пример добавления товаров к сделке:

```bash
curl -X POST "http://127.0.0.1:8000/sheets/marketplace" \
  -H "Content-Type: application/json" \
  -d '{
    "data": {
      "lead_id": 123456,
      "items": [
        {"id": 1001, "quantity": 2},
        {"id": 1002, "quantity": 0}
      ]
    }
  }'
```

Пример уведомления Яндекс Маркета:

```bash
curl -X POST "http://127.0.0.1:8000/market/new_order/notification" \
  -H "Content-Type: application/json" \
  -d '{"orderId": 123456789}'
```

### Переходы в Telegram и MAX

| Метод и маршрут | Входные данные | Результат и побочные эффекты | Основные ошибки |
|---|---|---|---|
| `GET /telegram` | Необязательные query/cookie: UTM-метки, `yclid`, `cm_id`, `block` | Сохраняет переход в БД и перенаправляет с HTTP 302 на `TELEGRAM_BOT_URL?start=<record_id>` | Ошибка записи в БД |
| `GET /max` | Те же параметры | Сохраняет переход в БД и перенаправляет с HTTP 302 на `MAX_BOT_URL?start=<record_id>` | Ошибка записи в БД |

Поддерживаемые UTM-параметры: `utm_source`, `utm_medium`, `utm_campaign`, `utm_content`, `utm_term`.

Пример:

```bash
curl -i "http://127.0.0.1:8000/telegram?utm_source=yandex&utm_medium=cpc&utm_campaign=summer&yclid=test-click-id"
```

### Получение UTM-меток

`GET /get_utm/{record_id}` возвращает сохранённые `utm_source`, `utm_medium`, `utm_campaign`, `utm_content`, `utm_term` и `yclid`.

Параметры:

- `record_id` — ID строки `education_visits`;
- `utm_token` — значение `GET_UTM_TOKEN`.

Возможные ответы:

- `200` — данные найдены;
- `401` — передан неверный токен;
- `404` — запись отсутствует или не содержит UTM/`yclid`;
- `500` — `GET_UTM_TOKEN` не настроен.

```bash
curl "http://127.0.0.1:8000/get_utm/42?utm_token=replace_me"
```

### Коммерческие предложения

| Метод и маршрут | Входные данные | Результат и побочные эффекты | Основные ошибки |
|---|---|---|---|
| `GET /kp` | Query: `lead_id` | Получает сделку и товары из amoCRM и возвращает HTML коммерческого предложения | `404`, если нет элементов каталога; `400`, если не найден ID каталога; ошибки данных amoCRM |
| `GET /kp/pdf` | Query: `lead_id` | Формирует то же предложение в PDF и удаляет временный файл после отправки | Ошибки `/kp`; `500` при ошибке рендеринга PDF |
| `GET /kp/service` | Нет | Возвращает статический HTML-шаблон предложения по услугам | Ошибка загрузки шаблона |
| `GET /kp/partner` | Query: JSON-строка `context`; также поддерживается опечатка `contex` | Рендерит переданный контекст шаблоном основного КП, возвращает PDF и удаляет временный файл | `400` при отсутствующем, невалидном или не объектном JSON; `500` при ошибке рендеринга |
| `GET /kp/assets/img01.png` | Нет | Внутреннее изображение шаблона; исключено из OpenAPI | `404`, если файл отсутствует |
| `GET /kp/assets/logo.png` | Нет | Внутренний логотип шаблона; исключён из OpenAPI | `404`, если файл отсутствует |
| `GET /kp/assets/montage_image.webp` | Нет | Внутреннее изображение шаблона; исключено из OpenAPI | `404`, если файл отсутствует |

Примеры HTML и PDF по сделке:

```bash
curl "http://127.0.0.1:8000/kp?lead_id=123456" -o kp.html
curl "http://127.0.0.1:8000/kp/pdf?lead_id=123456" -o kp.pdf
```

Для `/kp/partner` значение `context` должно быть URL-кодированной JSON-строкой. Каноническое имя параметра — `context`; вариант `contex` оставлен для совместимости с существующими вызовами.

```bash
curl --get "http://127.0.0.1:8000/kp/partner" \
  --data-urlencode 'context={"lead_id":"partner-42","proposal_number":"partner-42","client_name":"Тестовый клиент","products":[],"total_amount":0,"total_discount":0}' \
  -o partner-kp.pdf
```

Шаблон допускает расширенный контекст: сведения о клиенте и менеджере, список товаров, суммы, проект, даты и дополнительные блоки содержимого. Поля, необходимые для конкретного оформления, определяются шаблоном `services/templates/hite_pro_kp.html`.

## Основные потоки данных

### Заказ Яндекс Маркета

1. Яндекс Маркет отправляет `orderId` на `/market/new_order/notification`.
2. Сервис запрашивает сведения о заказе и покупателе через Partner API.
3. В amoCRM создаются контакт и сделка.
4. Состав заказа и адрес добавляются к сделке в виде примечания.
5. Исходный payload отправляется администратору в Telegram.

### Аналитика

1. Защищённый маршрут принимает `request_id` и токен.
2. FastAPI возвращает подтверждение и запускает фоновую задачу.
3. Задача загружает сделки и покупателей из amoCRM.
4. Данные преобразуются в JSON и отправляются на настроенный вебхук Google Sheets.
5. Результат записывается в журнал приложения.

### Коммерческое предложение

1. `/kp` получает сделку, ответственного менеджера и элементы каталога из amoCRM.
2. Сервис рассчитывает цены, скидки, доставку и итоговые суммы.
3. Jinja2 формирует HTML по шаблону.
4. `/kp/pdf` дополнительно запускает Chrome/Chromium в headless-режиме.
5. Временный PDF удаляется после отправки ответа.

## Типовые проблемы

### Приложение не запускается из-за переменной окружения

Проверьте наличие всех обязательных переменных из таблицы выше. `environs` прерывает загрузку конфигурации, если обязательное значение отсутствует.

### Ошибки авторизации amoCRM

- проверьте поддомен, client ID, client secret и redirect URL;
- убедитесь, что access/refresh token актуальны;
- разрешите процессу обновлять `.env`;
- не запускайте несколько экземпляров, конкурентно записывающих один локальный `.env`.

### Аналитика возвращает ошибку конфигурации

Для `/analyze` нужны `GOOGLE_SHEETS_WEBHOOK_URL` и `GOOGLE_SHEETS_TOKEN`. Для аналитики покупателей дополнительно настройте `GOOGLE_SHEETS_CUSTOMERS_WEBHOOK_URL`.

### Не создаётся SQLite-файл или таблица

Убедитесь, что рабочий каталог доступен на запись. Для другой СУБД проверьте SQLAlchemy URL и наличие соответствующего драйвера в окружении.

### Не формируется PDF

- установите Google Chrome или Chromium;
- убедитесь, что бинарный файл доступен через `PATH` или находится в стандартном системном каталоге;
- запускайте сервис из корня проекта, чтобы шаблоны и изображения разрешались корректно;
- проверьте права на создание каталога `services/tmp_pdf` и временных файлов.

### Тесты завершаются с `ModuleNotFoundError`

Активируйте виртуальное окружение и повторно установите зависимости из `requirements.txt`. Например, модуль `dotenv` предоставляется пакетом `python-dotenv`.

## Безопасность

- не публикуйте `.env`, токены и реальные webhook URL;
- ограничьте внешнюю доступность маршрутов, у которых нет встроенной авторизации;
- учитывайте, что токены `/analyze` и `/get_utm` передаются в query string и могут попасть в access-логи;
- используйте HTTPS и фильтрацию логов в production;
- примеры в этом README содержат только фиктивные значения.
